# Audit Log Agent / 审计日志 Agent

> 所有 Agent 操作生成审计日志存储在 Walrus，使用 Seal 加密敏感部分，满足合规和审计需求

## 解决什么问题

AI Agent 在执行任务时缺乏可追溯的操作日志，无法满足企业合规审计要求。Agent 处理的敏感数据（用户信息、交易细节）在日志中需要保护隐私，但又要保证审计时可以验证。本方案为 Agent 的每一步操作生成结构化审计日志，存储在 Walrus 确保不可篡改，通过 Seal 加密敏感字段实现隐私与可验证性的平衡。

## 核心功能

- 全量操作日志：自动捕获 Agent 的每个决策、工具调用、数据访问操作
- 敏感数据加密：使用 Seal 对日志中的敏感字段加密，仅授权审计员可解密
- 合规报告生成：定期自动生成符合 SOC2/GDPR 标准的合规审计报告

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储加密审计日志和合规报告 |
| Sui 对象模型 | 链上记录日志批次哈希、审计策略、授权列表 |
| Seal 加密 | 加密日志中的敏感字段（PII、交易数据等） |
| Walrus 不可篡改 | 利用 Walrus 确保日志存储后无法被修改 |

## 实现方案

### Agent / AI 部分

- 拦截 Agent 作为中间件自动捕获所有 Agent 操作的输入输出
- 脱敏 Agent 自动识别日志中的敏感信息并标记加密字段
- 加密 Agent 使用 Seal 对标记字段进行加密处理
- 审计 Agent 定期扫描日志，检查合规违规和异常行为
- 报告 Agent 自动生成合规审计报告，包含统计摘要和异常事件

### Walrus 集成

- 原始日志以 `audit/{agent_id}/logs/{date}/{batch}.json.enc` 加密存储
- 日志索引以 `audit/{agent_id}/index/{date}.json` 存储明文摘要
- 合规报告以 `audit/{agent_id}/reports/{period}.pdf` 存储
- 审计策略以 Sui 链上对象存储，定义敏感字段规则和加密策略
- 授权列表以 Sui 链上管理，控制谁可以解密查看敏感日志

### 前端 / 后端

- 前端：React 审计面板，支持日志查询、解密预览、合规报告查看
- 后端：Python 日志采集服务 + Rust 高性能加密模块
- 关键页面：日志搜索页、合规仪表盘、异常告警页、报告导出页

## 技术架构

Agent 执行操作 → 拦截 Agent 捕获输入输出 → 脱敏 Agent 标记敏感字段 → 加密 Agent 使用 Seal 加密敏感数据 → 日志批次写入 Walrus → Sui 链上记录批次哈希 → 审计 Agent 定期扫描 → 检测异常行为触发告警 → 报告 Agent 生成合规报告 → 授权审计员通过 Sui 验证身份 → 解密查看敏感日志。
