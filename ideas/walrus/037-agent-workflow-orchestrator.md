# Agent Workflow Orchestrator / Agent 工作流编排器

> 可视化编排多 Agent 工作流，Walrus 作为中间数据存储和状态协调中心

## 解决什么问题

多个 AI Agent 之间的协作缺乏可视化的编排工具，工作流的定义和调试过程复杂且不透明。Agent 之间的中间数据传递没有可靠的持久化方案，一旦某个环节失败整个流程需要重头开始。本方案提供拖拽式的工作流编排界面，以 Walrus 作为中间数据总线，实现 Agent 工作流的可视化定义、监控和调试。

## 核心功能

- 可视化工作流编辑器：拖拽式设计 Agent 工作流，定义数据流向和分支条件
- 状态持久化：每个 Agent 节点的输入输出中间结果存储在 Walrus，支持断点续跑
- 实时监控面板：展示工作流执行进度、各节点状态、数据流转可视化

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储工作流定义、Agent 中间结果、执行日志 |
| Sui 对象模型 | 链上记录工作流注册、执行状态、权限控制 |
| Sui 事件机制 | 工作流节点完成时触发事件通知下游节点 |
| Seal 加密 | 加密工作流中的敏感中间数据 |

## 实现方案

### Agent / AI 部分

- 编排 Agent 解析工作流 DAG 图，按拓扑顺序调度各节点执行
- 路由 Agent 根据条件分支决定数据流向哪个下游节点
- 重试 Agent 检测节点失败并自动重试或降级处理
- 监控 Agent 收集各节点执行指标，生成性能报告
- 调试 Agent 在工作流异常时自动分析日志定位问题节点

### Walrus 集成

- 工作流定义以 `workflow/{id}/definition.json` 存储在 Walrus
- 节点输入以 `workflow/{id}/nodes/{node_id}/input.json` 存储
- 节点输出以 `workflow/{id}/nodes/{node_id}/output.json` 存储
- 执行日志以 `workflow/{id}/logs/{timestamp}.json` 存储
- 工作流元信息以 Sui 链上对象存储，记录状态和触发条件

### 前端 / 后端

- 前端：React Flow 可视化编辑器 + 实时监控面板
- 后端：TypeScript 编排引擎 + Python Agent 运行时
- 关键页面：工作流编辑器、执行监控页、日志查看页、模板库页

## 技术架构

用户在编辑器中拖拽设计工作流 → 定义保存到 Walrus → 触发执行 → 编排 Agent 解析 DAG → 按拓扑序调度节点 → Agent 节点从 Walrus 读取上游输出 → 执行任务 → 结果写入 Walrus → 触发 Sui 事件通知 → 路由 Agent 决定下游分支 → 循环直到所有节点完成 → 监控面板实时展示进度 → 异常时调试 Agent 自动分析。
