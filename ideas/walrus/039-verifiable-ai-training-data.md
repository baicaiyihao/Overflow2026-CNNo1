# Verifiable AI Training Data / 可验证 AI 训练数据

> 训练数据集存储在 Walrus 并哈希上链，确保 AI 训练数据的完整性、来源可追溯和防篡改

## 解决什么问题

AI 模型的训练数据来源不透明，无法验证模型是否使用了声明之外的数据集。训练数据在存储和传输过程中可能被篡改，导致模型行为不可预测。数据版权和合规性问题难以追溯。本方案将训练数据集存储在 Walrus，数据哈希和元数据上链，建立从数据采集到模型训练的完整可信证据链。

## 核心功能

- 数据集确权：训练数据上传 Walrus 后自动生成链上哈希指纹，确权数据集版本
- 来源追溯：记录每个数据样本的采集来源、时间戳、处理过程，支持全链路追溯
- 完整性验证：任何人可通过链上哈希验证 Walrus 中的数据是否被篡改

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储训练数据集文件（CSV、JSON、Parquet 等） |
| Sui 对象模型 | 链上记录数据集元信息、哈希、版本号、许可协议 |
| Seal 加密 | 加密敏感训练数据，仅授权方可访问 |
| Sui 事件机制 | 数据集注册、更新、验证时触发链上事件 |

## 实现方案

### Agent / AI 部分

- 采集 Agent 从多个数据源自动收集训练数据样本
- 验证 Agent 对每个数据样本进行质量检查（格式、完整性、去重）
- 哈希 Agent 计算数据集的 Merkle 树哈希，生成可验证证明
- 追溯 Agent 构建数据样本到来源的完整追溯链
- 合规 Agent 检查数据集是否符合使用许可和隐私法规

### Walrus 集成

- 原始数据集以 `datasets/{id}/raw/data.parquet` 格式存储
- 数据集元信息以 `datasets/{id}/metadata.json` 存储
- Merkle 证明以 `datasets/{id}/proofs/merkle.json` 存储
- 数据来源日志以 `datasets/{id}/provenance.json` 存储
- 数据集注册信息以 Sui 链上对象存储，包含哈希和版本

### 前端 / 后端

- 前端：React 数据集管理界面，支持上传、验证、追溯
- 后端：Python 数据处理服务 + Move 合约
- 关键页面：数据集上传页、验证结果页、追溯链页、合规报告页

## 技术架构

数据提供者上传训练数据 → 采集 Agent 收集并预处理 → 验证 Agent 质量检查 → 合规 Agent 许可审查 → 数据集写入 Walrus → 哈希 Agent 计算 Merkle 根 → Sui 链上注册数据集对象（含哈希） → 追溯 Agent 记录来源链 → 用户发起验证请求 → 从 Walrus 下载数据 → 重算哈希与链上记录比对 → 返回验证结果。
