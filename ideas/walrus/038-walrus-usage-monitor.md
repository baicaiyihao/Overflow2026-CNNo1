# Walrus Usage Monitor / Walrus 使用量监控

> 监控 Agent 的 Walrus 存储使用量、访问频率和成本预估，帮助优化存储策略

## 解决什么问题

Agent 持续向 Walrus 写入数据却缺乏用量感知，容易造成存储浪费和成本失控。开发者和用户无法直观了解哪些 Agent 写入了多少数据、访问频率如何、何时需要清理。本方案提供全面的 Walrus 使用量监控面板，实时追踪存储消耗、访问模式和成本趋势，帮助用户优化存储策略。

## 核心功能

- 实时用量追踪：监控每个 Agent 的 Blob 写入量、读取次数、存储占用
- 成本预估仪表盘：基于当前用量趋势预估未来存储成本，设置预算告警
- 智能清理建议：AI 分析 Blob 访问模式，推荐可清理或归档的低价值数据

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 读取 | 读取 Blob 元信息（大小、创建时间、过期时间） |
| Walrus Blob 存储 | 存储用量统计数据和分析报告 |
| Sui 对象模型 | 链上记录 Blob 归属、使用配额、费用信息 |
| Sui 链上数据 | 查询 Blob 相关的链上交易计算用量 |

## 实现方案

### Agent / AI 部分

- 采集 Agent 定期从 Sui 链上和 Walrus 网关收集用量原始数据
- 聚合 Agent 按时间维度（小时/天/周）聚合用量数据，生成统计摘要
- 分析 Agent 识别访问模式（冷热数据分布、增长趋势、异常突增）
- 预测 Agent 基于历史数据预测未来用量和成本
- 建议 Agent 生成数据清理和存储优化建议

### Walrus 集成

- 原始用量数据以 `metrics/{agent_id}/raw/{date}.json` 存储
- 聚合报告以 `metrics/{agent_id}/summary/{period}.json` 存储
- 趋势分析以 `metrics/{agent_id}/trends.json` 存储
- 清理建议以 `metrics/{agent_id}/suggestions.json` 存储
- 用量配额以 Sui 链上对象管理，支持阈值告警

### 前端 / 后端

- 前端：React + Recharts 数据可视化仪表盘
- 后端：Go 高性能数据采集和聚合服务
- 关键页面：用量概览页、Agent 详情页、成本预估页、清理建议页

## 技术架构

定时触发采集 Agent → 从 Sui 链上获取 Blob 相关交易 → 从 Walrus 网关获取 Blob 元信息 → 聚合 Agent 按维度汇总数据 → 分析 Agent 识别模式 → 预测 Agent 计算趋势 → 统计数据写入 Walrus → 建议 Agent 生成优化方案 → 前端仪表盘实时展示 → 超阈值时触发告警通知。
