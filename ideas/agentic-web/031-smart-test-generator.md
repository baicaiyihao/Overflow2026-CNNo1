# Smart Test Generator / 智能测试生成器

> 分析 Move 合约接口，AI 自动生成测试用例，覆盖边界条件

## 解决什么问题

Move 合约的测试覆盖率普遍不足，开发者往往只编写正常路径的测试，忽略边界条件和异常情况。手动编写全面的测试用例耗时且容易遗漏关键场景，特别是涉及对象所有权转移和权限变更的复杂测试。本项目通过 AI 分析 Move 合约接口和业务逻辑，自动生成覆盖全面的测试用例，包含边界条件和异常路径，提升 Sui 生态合约质量。

## 核心功能

- 解析 Move 合约模块接口，AI 自动生成测试函数，覆盖正常路径、边界条件和异常回滚
- 针对对象能力约束生成专项测试：验证 Store/Key/Drop 能力的正确使用和越权拒绝
- 生成测试数据工厂：自动创建模拟对象、Token 和账户，支持测试场景复用

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | 理解对象创建、转移和销毁路径，生成对应测试用例 |
| 能力系统 | 针对能力约束生成正向和反向测试（验证能力存在/缺失的行为） |
| 链上数据 | 读取已部署合约的实际状态生成回归测试 |
| PTB | 生成多步交互的集成测试场景（如「创建对象 → 转移 → 修改 → 验证」） |

## 实现方案

### 智能合约 (Move)
- **`test_registry` 模块**：定义 `TestSuite` 对象，字段包括 `package_id: ID`、`module_name: String`、`test_count: u64`、`coverage_score: u64`、`creator: address`
- **`coverage_report` 模块**：`record_coverage(suite: &mut TestSuite, branch_covered: u64, branch_total: u64, timestamp: u64)` 记录覆盖率数据
- **`test_template` 模块**：存储常用测试模式的 Move 代码模板，AI 可引用和组合
- 关键数据结构：`TestCase { name: String, target_function: String, category: u8, assertion_count: u64 }`

### 前端 / 后端
- **前端**：VS Code Webview 面板，展示生成的测试代码、覆盖率报告、测试矩阵
- **后端**：Python Agent 服务，Move AST 解析、测试策略规划、代码生成器
- 关键功能：一键生成测试、覆盖率可视化、测试编辑和导出、回归测试管理

### AI / Agent 部分
- 接口分析：解析 Move 函数签名，提取参数类型、能力约束和返回值
- 测试策略：基于等价类划分和边界值分析生成测试计划，AI 补充业务逻辑级测试
- 代码生成：GPT-4o 根据 Move 测试框架语法生成完整测试代码
- Agent 工作流：合约解析 → 接口提取 → 测试策略规划 → 测试代码生成 → 编译验证 → 覆盖率分析

## 技术架构

Agent 服务解析 Move 合约 AST，提取所有公共函数签名和对象交互模式。测试策略引擎基于等价类划分和边界值分析生成测试计划矩阵。GPT-4o 根据 Move 测试框架（`#[test]`、`#[expected_failure]`）语法生成测试代码。生成的测试自动编译验证，失败的测试由 AI 修复后重新验证。覆盖率分析器统计分支覆盖率，识别未覆盖路径后补充测试。`TestSuite` 对象记录测试结果和覆盖率上链。
