# Withdrawal Limiter Monitor / 提款限制器监控

> 追踪 PLP 提款限制器令牌桶状态，为 LP 提供流动性预警

## 解决什么问题

Predict 的 PLP（Predict Liquidity Provider）金库使用提款限制器（令牌桶算法）控制资金流出速率，防止短期内大量赎回导致金库流动性枯竭。但 LP 缺乏实时工具查看当前提款限制器状态，可能导致提交赎回请求时被拒绝或延迟。Withdrawal Limiter Monitor 实时追踪令牌桶状态，为 LP 提供流动性预警和最优提款时间建议。

## 核心功能

- 实时追踪 PLP 提款限制器令牌桶的剩余容量和恢复速率
- 预测下次可提款时间窗口和最大可提款金额
- 历史提款模式分析：高峰期、低谷期、平均等待时间
- 自定义告警：令牌桶恢复到指定容量时通知用户

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::supply | 理解 PLP 供应和提款机制 |
| 提款限制器状态 | 核心监控对象 |
| Sui 链上状态 | 读取令牌桶参数和历史 |
| PredictManager | 追踪 LP 仓位和提款请求 |

## 实现方案

### Bot / Keeper / 服务端
- 令牌桶模拟器：根据链上参数实时计算令牌桶剩余容量
- 链上状态读取器：定期读取提款限制器的当前状态
- 时间预测器：根据恢复速率预测下次可提款窗口
- 提款分析器：统计历史提款模式，识别高峰低谷
- 告警引擎：令牌桶恢复到阈值时触发通知

### 前端 / 后端
- 技术栈：React + Node.js + Redis
- 令牌桶可视化：当前容量、恢复速率、预计满桶时间
- 提款时间建议：最佳提款窗口标注
- 历史提款统计：每日/每周提款量趋势

## 技术架构

链上状态读取器定期获取令牌桶参数 → 令牌桶模拟器实时计算剩余容量 → 时间预测器推算可提款窗口 → 达到阈值时告警引擎通知 → 前端展示令牌桶状态和提款建议。
