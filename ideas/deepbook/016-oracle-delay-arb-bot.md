# Oracle Delay Arb Bot / 预言机延迟套利机器人

> 利用 Predict 预言机更新延迟，在价格更新前后抢交易赚取无风险利润

## 解决什么问题

Predict 的 OracleSVI 更新存在可观测的延迟窗口，链上预言机价格与 Binance 实时价格之间有时间差。套利者可以利用这个延迟，在价格实际更新到链上之前，预测更新方向并提前布局仓位。Oracle Delay Arb Bot 自动监控链下价格变动并预测预言机更新方向，在更新前后执行套利交易。

## 核心功能

- 实时监控 Binance BTC 价格与链上 OracleSVI 的偏差
- 预测预言机下一次更新方向，提前买入被低估的区间
- 在预言机更新后立即对冲或平仓，锁定价差利润
- 多到期日并行扫描，选择最优套利窗口

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| OracleSVIUpdated 事件 | 监控预言机更新频率和延迟 |
| predict::mint | 在价格更新前买入被低估区间 |
| predict::redeem_permissionless | 更新后赎回获利仓位 |
| PredictManager | 管理套利仓位和资金 |

## 实现方案

### Bot / Keeper / 服务端
- 价格偏差计算器：Binance WebSocket 与链上 OracleSVI 持续对比
- 更新预测模型：基于历史更新间隔和价格走势预测下次更新方向
- 快速执行器：检测到阈值偏差时通过 PTB 原子提交 mint 交易
- 延迟监控：统计平均更新延迟，动态调整策略参数
- 风控模块：最大单笔暴露、延迟回归均值保护

### 前端 / 后端
- 技术栈：React + Python FastAPI + Redis
- 实时价格偏差面板（链下 vs 链上）
- 套利机会时间线和历史 PnL
- 延迟统计和预言机更新频率图表

## 技术架构

Binance WebSocket 推送实时价格 → 偏差计算器对比链上 OracleSVI → 超过阈值时生成信号 → 快速执行器通过 predict::mint 提交交易 → 预言机更新后 redeem 平仓 → 前端实时展示偏差和套利状态。
